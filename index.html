FortiGate Infrastructure

System&Network&Setting

接口下 Administrative Access FTM 当Fortitoken连接时需要打开
接口下 Administrative Access FMG-Access 只用来连接Fortimanage时需要打开
接口下 Administrative Access Security Fabiric 用来连接 Fortinet 各个产品之间的联动、遥测（Telemetry）和拓扑同步

Firewall Policies and NAT

防火墙策略配置下，当不知道Policy ID使用了多少，可以使用edit 0来让系统自动提供一个可用ID
防火墙安全策略中，通过右边选取BY Sequence的方式，可以查看策略匹配顺序情况
NAT overload 用于办公环境，one to one用于dmz服务器区
VIP 开启Port Forwarding，外部地址端口映射到内部主机
match-vip 当VIP上一条拒绝所有的地址访问，但允许访问特定地址，需要在上一条策略中开启禁用匹配VIP意味着FortiGate在政策评估时不要同时检查VIP。注意，match-vip 设置仅在防火墙策略作设置为 DENY 时可用
arp-reply 默认是开启状态，当在上游ISP router与Forit之间，ISP将公共子网分配给客户，但是没有该子网的静态路由，以便于生成ARP请求。已解析子网中任意地址的MAC地址

Routing

同时可以根据FQDN、internet Service来配置静态明细路由

Authentication

FortiToken软件目前没法迁移，比如Foti100E上200个token，不能将软token迁移到forti200E
FAC 身份认证服务器进行绑定

Fortinet Single Sign-On(FSSO)

FSSO agent安装,需要有一个可以读取日志的权限
combination用户排除，不想对某些用户进行无感知认证

Antivirus & Web Filtering

对延时敏感开流模式，对安全检查要求高的开代理模式

Certificate

subject or san对返回的SNI字段进行验证，通过sni中获得他的url
没有导入用户的证书，建议客户导入证书
老的系统如win7 没有sni识别标记，需要服务器数字证书进行签名和验证
web过滤和用户验证 会遇到SSL证书检测
SSL深度检测，forti启用到类似代理的重用用自己的key进行加密/解密 （会有延迟）
分为两类检测：流模式检测和深度检测
开启7层检测，需要用户有证书，必须导入，换证书需要进行自定义克隆进行修改，需深入了解，查看RFC5280

IPS adn Application Control

上面宽泛，下面明细，可以上面对特定的进行证书检测，最下面对所有做深度检测
IPS引擎：AV，appcontrol（始终基于流模式），自定义优先级高于categories分类优先级
进行抓包，通过Forti团队再去写对应的应用特征签名

SSL VPN

FortiOS 7.6.3之后没有免费的SSLVPN，要用使用收费的Forticlient VPN，直接用IPsec
https://community.fortinet.com/t5/FortiGate/Technical-Tip-SSL-VPN-support-on-FortiGate-models/ta-p/359125

IPsec VPN

华为ipsec vpn 对接请使用ikev2版本
add route ,明细路由会自动入录
DDNS 点对点的环境，动态公网IP（至少有一个是静态地址）DNS服务器一定要配置
第三方深信服 aggressive模式可能更好一些，main同厂商更好

SDWAN

FortiOS 7.2.x策略路由会替换掉直连路由，要关闭路由不转发
https://handbook.fortinet.com.cn/

Security Fabric

至少一个上游root设备，一个下游设备, 同时root设备与fortgate cloud，fortianlizer或 fortianlizer cloud进行对接
通过内网或IPsec VPN环境去做，一般不会通过公网地址去做
同步对象后，只能通过root防火墙去进行修改
Fortimanage设备比较多的情况推荐使用

High Availability

FGCP协议将两台设备进行高可用性
HA要求：型号一样，固件版本，许可（都需要）许可不同时已许可少的显示
会话同步线，建议多增加一条单独同步线
端口优先级越高,设定的ha接口越优先
HA环境中，带外特定端口建议都做一下，便于单独登陆设备。而不是只能登陆一台主设备，方便运维考量
通过主防火墙，可以通过ha manage进行跳转，注意：只有cli界面可以
防火墙<->核心，linkmonitor ip
通过link-monitor，可以单独对链路来做切换
升级会丢包，备机，业务切到备机，再升级主机。大于5分钟就存在备机上。小于5分钟还需要切换一次

Diagnostics and Troubleshooting

防火墙本身不是日志监控的设备，只是日志监控的一个功能。如果需要看到更多信息建议通过fortianlizer或搭建第三方开源的产品例如zabbix

hexo 建站

发表于 2025-12-05 分类于 springboot

Hexo 使用 Node.js 和 Markdown，非常适合快速搭建内容丰富的技术博客。

步骤一：环境准备
您需要确保您的系统安装了以下工具：

Node.js (推荐 LTS 版本)

Git

步骤二：安装 Hexo
打开您的终端或命令行工具，运行以下命令安装 Hexo：

Bash

npm install -g hexo-cli
步骤三：初始化项目
选择一个目录，创建您的博客项目：

Bash

blog 是您的项目名称

hexo init blog
cd blog

安装依赖

npm install
步骤四：选择并配置主题 (Theme)
这是实现图片中科技感外观的关键一步。

  1. 寻找主题
    您需要寻找一个具有简洁、科技或代码风格的主题。以下是一些热门主题，您可以在 Hexo 官方主题库中查看预览：

Next (v7/v8): 简洁、经典，容易自定义。

Volantis: 功能强大，设计现代，适合复杂博客。

Hexture: 扁平化设计，也比较适合技术博客。

  1. 安装主题（以 Next 为例）
    Bash

将主题克隆到 themes 文件夹下

git clone https://github.com/next-theme/hexo-theme-next themes/next
3. 启用主题
修改博客根目录下的主配置文件 _config.yml，找到 theme 字段并修改为您安装的主题名称：

YAML

文件: blog/_config.yml

theme: next
步骤五：实现图片中的核心功能

  1. 科技感的星空背景
    您可以修改主题的配置文件（通常是 themes/next/_config.yml 或其他主题的配置）来启用背景图片或特效。

背景图片： 将您准备的星空图片放到主题的静态资源目录（例如 themes/next/source/images/），然后在 CSS 中引用它作为博客主页或文章页的背景。

  1. 文章侧边目录 (TOC)
    Hexo 主题通常内置了自动生成 TOC 的功能。

配置启用： 在主题的配置文件中，查找关于 TOC 或 table_of_contents 的设置，确保它被设置为 true 或启用状态。

写作要求： 您在 Markdown 文章中需要正确使用标题等级（# 一级标题，## 二级标题，### 三级标题）。TOC 会自动抓取这些标题来生成侧边栏链接。

  1. 标签（Tag）显示
    在您的文章文件（位于 source/_posts/ 目录下）的开头，通过 Front-matter 来定义标签：

Markdown

title: 监控利器之 grafana 与 mysql 整合实战
date: 2021-06-20
tags: [apm, grafana, mysql, 监控] # 这里定义了标签

序言

… 您的文章内容 …
步骤六：本地预览与部署

本地预览

在项目根目录运行以下命令，即可在本地浏览器中预览效果：

Bash

hexo clean # 清除缓存
hexo generate # 简称 hexo g，生成静态文件
hexo server # 简称 hexo s，启动本地服务器
访问浏览器地址 http://localhost:4000 即可查看您的博客。

卸载冲突插件（可选，但推荐）

如果您的博客安装了其他类似的图片路径处理插件（例如 hexo-asset-image），为避免冲突，我们先移除它。

在博客根目录下运行：

Bash

npm uninstall hexo-asset-image

安装推荐插件

我们安装一个专门处理 asset_img 标签的插件，它可以更好地兼容 NexT 主题。

在博客根目录下运行：

Bash

npm install hexo-asset-img –save
（注意：这个插件名是 hexo-asset-img，与上面卸载的 hexo-asset-image 略有不同，但功能类似且更为稳定。）

再次确认配置和使用方法

A. 确认配置：

站点配置文件 (_config.yml - 根目录) 中：

YAML

post_asset_folder: true
文件结构：

.
└── source/
└── _posts/
├── MyNewPost.md <– 文章文件
└── MyNewPost/ <– 资源文件夹 (与文章文件同名)
└── demo.jpg <– 图片文件
B. 确认语法： 在 MyNewPost.md 中，引用时只需写文件名，不需要写路径：

Markdown

清理并重新生成

在终端执行：

Bash

hexo clean
hexo g
hexo s

部署到互联网

确保“分类”总页面已开启

首先，你必须确保 Hexo 的分类功能已经初始化（如果你之前只创建了 springboot 或 linux 的独立页面，可能漏了这一步）。

在博客根目录打开终端，输入：

Bash

hexo new page categories
打开生成的 source/categories/index.md 文件，内容修改如下：

YAML

title: categories
date: 2025-12-04 14:25:53
type: “categories” <– 这一行最重要，不能少

（注意：此时你访问 /categories/ 是可以看到总分类页的，但 /categories/project/ 依然会报错，直到你做了第二步。）

第二步：写一篇属于该分类的文章（关键）
Hexo 需要至少一篇文章标记为 project，它才会自动生成 /categories/project/ 这个路径的 HTML 文件。

新建一篇文章（或者打开一篇现有的）：

Bash

hexo new “我的项目演示”

打开这篇文章（在 source/_posts/ 下），在顶部的 Front-matter（头部配置区）添加 categories：

YAML

title: 我的项目演示
date: 2025-12-04 15:00:00
categories:
- project <– 这里必须和你在菜单里填写的路径名一致
tags:
- java

这是我的第一个项目文章内容…
第三步：清理并重启
修改完后，必须重新生成一遍才能生效。

在终端执行：

Bash

hexo clean
hexo g
hexo s
现在，再次点击菜单里的 “project”，或者访问 http://localhost:4000/categories/project/，你应该能看到页面了，页面里会列出刚才那篇“我的项目演示”文章。

静态博客最常用且免费的部署方式是使用 GitHub Pages 或 Vercel/Netlify。

安装部署插件：

Bash

npm install hexo-deployer-git –save
配置部署信息： 修改主配置文件 _config.yml：

YAML

文件: blog/_config.yml

deploy:
type: git
替换为您的 GitHub Page 仓库地址
repo: <您的GitHub Pages仓库URL>
branch: main # 或 gh-pages
一键部署：

Bash

hexo deploy # 简称 hexo d

配置 Git 用户身份

您只需按照提示，在您的 Windows 命令行（PowerShell 或 Command Prompt）中运行两条命令，设置您的全局 Git 用户名和电子邮件地址即可。

设置用户电子邮件： 将下面的 “you@example.com“ 替换为您自己的邮箱地址。

Bash

git config –global user.email “your_email@example.com“
设置用户名称： 将 “Your Name” 替换为您的名字或您希望显示的用户名。

Bash

git config –global user.name “Your Name”

h3c无线AC分层AP结合LDAP无线认证

发表于 2025-12-04 分类于 project

架构如上图所示，ac用于对接ad域服务器，dhcp服务位于三层交换机

H3C官方配置教程：https://www.h3c.com/cn/d_202108/1436778_30005_0.htm

配置注意事项

· 配置AP的序列号时请确保该序列号与AP唯一对应。

· 如果本地Portal Web服务器提供的缺省认证页面文件需要更新，需要undo default-logon-page后重新配置，否则新页面不会生效。

· 配置路由，保证启动Portal之前各设备之间的路由可达。

· 请提前编辑好认证页面，保存为abc.zip，并上传到AC存储介质的根目录

所需要用到的vlan：

1、vlan 50用于各网络设备的管理及静态路由数据的转发，ldap服务器也可以接入到这个vlan，如果ldap服务器有其他网段，就在ac新建一个相应的网段

2、vlan172 用于无线网络dhcp使用

3、vlan 100 用于AP将获取该IP地址与AC建立CAPWAP隧道

3.3 配置步骤
3.3.1 配置AC
(1) 配置AC接口

创建VLAN 100及其对应的VLAN接口，并为该接口配置IP地址。AP将获取该IP地址与AC建立CAPWAP隧道。

system-view

[AC] vlan 100

[AC-vlan100] quit

[AC] interface vlan-interface 100

[AC-Vlan-interface100] ip address 2.2.1.1 24

[AC-Vlan-interface100] quit

创建VLAN 172及其对应的VLAN接口，并为该接口配置IP地址。Client将使用该VLAN接入无线网络。（也就是无线网络的ip段）

[AC] vlan 172

[AC-vlan172] quit

[AC] interface vlan-interface 172

[AC-Vlan-interface172] ip address 172.88.0.254 24

[AC-Vlan-interface172] quit

配置AC与Switch相连的GigabitEthernet1/0/1接口的属性为Trunk，允许所有vlan通过。

[AC] interface gigabitethernet 1/0/1

[AC-GigabitEthernet1/0/1] port link-type trunk

[AC-GigabitEthernet1/0/1] port trunk permit vlan all

[AC-GigabitEthernet1/0/1] quit

(2) 配置LDAP方案

创建LDAP服务器ldap，并进入LDAP服务器视图。

[AC] ldap server ldap

配置具有管理员权限的用户DN。（我的ad域的域名为network.lan，里面的cn=users应该是ad域服务器的本地组，为了方便管理，我们稍后在ad域里面创建一个wifi的组，然后把wifi这个组加入域服务器的本地users组就可以了，之后域用户需要连接wifi的统一加入wifi这个群组就可以了）

[AC-ldap-server-ldap] login-dn cn=administrator,cn=users,dc=network,dc=lan

配置查询用户的起始目录。

[AC-ldap-server-ldap] search-base-dn dc=network,dc=lan

配置LDAP认证服务器的IP地址。（这里填你的ad域服务器ip）

[AC-ldap-server-ldap] ip 192.168.50.211

配置具有管理员权限的用户密码。（这里填域管理员密码，最好创建一个具有管理权限的用户只用于此设备使用）

[AC-ldap-server-ldap] login-password simple 123456

[AC-ldap-server-ldap] quit

创建LDAP方案ldap，并进入LDAP方案视图。

[AC] ldap scheme ldap

配置LDAP认证服务器。

[AC-ldap-ldap] authentication-server ldap

[AC-ldap-ldap] quit

创建ISP域ldap，并进入ISP域视图。

[AC] domain ldap

为Portal用户配置AAA认证方法为LDAP认证、不授权、不计费。（这里注意了，别tap补全命令填到不认证了，这会导致点登陆按钮没反应）

[AC-isp-ldap]authentication portal ldap-scheme ldap

[AC-isp-ldap] authorization portal none

[AC-isp-ldap] accounting portal none

指定ISP域ldap下的用户闲置切断时间为15分钟，闲置切断时间内产生的流量为1024字节。

[AC-isp-ldap] authorization-attribute idle-cut 15 1024

[AC-isp-ldap] quit

(3) 配置Portal认证

配置Portal Web服务器的URL为http://172.88.0.254/portal。(这里填AC无线网络段的vlan地址，也就是上面设置的172.88.0.254)

[AC] portal web-server newpt

[AC-portal-websvr-newpt] url http://172.88.0.254/portal

[AC-portal-websvr-newpt] quit

创建本地Portal Web 服务器，进入本地Portal Web服务器视图，并指定使用HTTP协议和客户端交互认证信息。

[AC] portal local-web-server http

配置本地Portal Web服务器提供的缺省认证页面文件为defaultfile.zip（设备的存储介质的根目录下必须已存在该认证页面文件，否则功能不生效，我的ac设备默认有三个认证网页模板，我用的默认defaultfile.zip这个）。

[AC–portal-local-websvr-http] default-logon-page defaultfile.zip

[AC–portal-local-websvr-http] quit

配置两条基于目的的Portal免认证规则，放行访问DNS服务器的流量。

[AC] portal free-rule 1 destination ip any udp 53

[AC] portal free-rule 2 destination ip any tcp 53

开启无线Portal客户端合法性检查功能。

[AC] portal host-check enable

(4) 配置无线服务

创建无线服务模板st1，并进入无线服务模板视图。（这里的服务模版是可以改名字的，你可以自定义你喜欢的名字）

[AC] wlan service-template st1

配置SSID为service。（这里的名称是你的wifi名称，可以自定义修改）

[AC-wlan-st-st1] ssid service

配置无线服务模板VLAN为172。（就是把模版加入无线段的vlan）

[AC-wlan-st-st1] vlan 172

使能直接方式的Portal认证。

[AC-wlan-st-st1] portal enable method direct

配置接入的Portal用户使用认证域为ldap。

[AC-wlan-st-st1] portal domain ldap

在服务模板上引用名称为newpt的Portal Web服务器作为用户认证时使用的Web服务器。（newpt是可以改名字的看你喜欢）

[AC-wlan-st-st1] portal apply web-server newpt

配置客户端数据报文转发位置为AC。（如果客户端数据报文的缺省转发位置与本配置相同，请跳过此步骤）

[AC–wlan-st-st1] client forwarding-location ac

配置身份认证与密钥管理模式为PSK模式，配置PSK密钥为明文字符串12345678。（这里密码就是你的wifi连接密码）

[AC-wlan-st-st1] akm mode psk

[AC-wlan-st-st1] preshared-key pass-phrase simple 12345678

配置加密套件为CCMP，安全信息元素为RSN。

[AC-wlan-st-st1] cipher-suite ccmp

[AC-wlan-st-st1] security-ie rsn

使能无线服务模板。

[AC-wlan-st-st1] service-template enable

[AC-wlan-st-st1] quit
