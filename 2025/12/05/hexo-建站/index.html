hexo 建站

发表于 2025-12-05 分类于 springboot

Hexo 使用 Node.js 和 Markdown，非常适合快速搭建内容丰富的技术博客。

步骤一：环境准备
您需要确保您的系统安装了以下工具：

Node.js (推荐 LTS 版本)

Git

步骤二：安装 Hexo
打开您的终端或命令行工具，运行以下命令安装 Hexo：

Bash

npm install -g hexo-cli
步骤三：初始化项目
选择一个目录，创建您的博客项目：

Bash

blog 是您的项目名称

hexo init blog
cd blog

安装依赖

npm install
步骤四：选择并配置主题 (Theme)
这是实现图片中科技感外观的关键一步。

  1. 寻找主题
    您需要寻找一个具有简洁、科技或代码风格的主题。以下是一些热门主题，您可以在 Hexo 官方主题库中查看预览：

Next (v7/v8): 简洁、经典，容易自定义。

Volantis: 功能强大，设计现代，适合复杂博客。

Hexture: 扁平化设计，也比较适合技术博客。

  1. 安装主题（以 Next 为例）
    Bash

将主题克隆到 themes 文件夹下

git clone https://github.com/next-theme/hexo-theme-next themes/next
3. 启用主题
修改博客根目录下的主配置文件 _config.yml，找到 theme 字段并修改为您安装的主题名称：

YAML

文件: blog/_config.yml

theme: next
步骤五：实现图片中的核心功能

  1. 科技感的星空背景
    您可以修改主题的配置文件（通常是 themes/next/_config.yml 或其他主题的配置）来启用背景图片或特效。

背景图片： 将您准备的星空图片放到主题的静态资源目录（例如 themes/next/source/images/），然后在 CSS 中引用它作为博客主页或文章页的背景。

  1. 文章侧边目录 (TOC)
    Hexo 主题通常内置了自动生成 TOC 的功能。

配置启用： 在主题的配置文件中，查找关于 TOC 或 table_of_contents 的设置，确保它被设置为 true 或启用状态。

写作要求： 您在 Markdown 文章中需要正确使用标题等级（# 一级标题，## 二级标题，### 三级标题）。TOC 会自动抓取这些标题来生成侧边栏链接。

  1. 标签（Tag）显示
    在您的文章文件（位于 source/_posts/ 目录下）的开头，通过 Front-matter 来定义标签：

Markdown

title: 监控利器之 grafana 与 mysql 整合实战
date: 2021-06-20
tags: [apm, grafana, mysql, 监控] # 这里定义了标签

序言

… 您的文章内容 …
步骤六：本地预览与部署

1. 本地预览

在项目根目录运行以下命令，即可在本地浏览器中预览效果：

Bash

hexo clean # 清除缓存
hexo generate # 简称 hexo g，生成静态文件
hexo server # 简称 hexo s，启动本地服务器
访问浏览器地址 http://localhost:4000 即可查看您的博客。

2. 卸载冲突插件（可选，但推荐）

如果您的博客安装了其他类似的图片路径处理插件（例如 hexo-asset-image），为避免冲突，我们先移除它。

在博客根目录下运行：

Bash

npm uninstall hexo-asset-image

2.1. 安装推荐插件

我们安装一个专门处理 asset_img 标签的插件，它可以更好地兼容 NexT 主题。

在博客根目录下运行：

Bash

npm install hexo-asset-img –save
（注意：这个插件名是 hexo-asset-img，与上面卸载的 hexo-asset-image 略有不同，但功能类似且更为稳定。）

2.2. 再次确认配置和使用方法

A. 确认配置：

站点配置文件 (_config.yml - 根目录) 中：

YAML

post_asset_folder: true
文件结构：

.
└── source/
└── _posts/
├── MyNewPost.md <– 文章文件
└── MyNewPost/ <– 资源文件夹 (与文章文件同名)
└── demo.jpg <– 图片文件
B. 确认语法： 在 MyNewPost.md 中，引用时只需写文件名，不需要写路径：

Markdown

2.3. 清理并重新生成

在终端执行：

Bash

hexo clean
hexo g
hexo s

3. 部署到互联网

确保“分类”总页面已开启

首先，你必须确保 Hexo 的分类功能已经初始化（如果你之前只创建了 springboot 或 linux 的独立页面，可能漏了这一步）。

在博客根目录打开终端，输入：

Bash

hexo new page categories
打开生成的 source/categories/index.md 文件，内容修改如下：

YAML

title: categories
date: 2025-12-04 14:25:53
type: “categories” <– 这一行最重要，不能少

（注意：此时你访问 /categories/ 是可以看到总分类页的，但 /categories/project/ 依然会报错，直到你做了第二步。）

第二步：写一篇属于该分类的文章（关键）
Hexo 需要至少一篇文章标记为 project，它才会自动生成 /categories/project/ 这个路径的 HTML 文件。

新建一篇文章（或者打开一篇现有的）：

Bash

hexo new “我的项目演示”

打开这篇文章（在 source/_posts/ 下），在顶部的 Front-matter（头部配置区）添加 categories：

YAML

title: 我的项目演示
date: 2025-12-04 15:00:00
categories:
- project <– 这里必须和你在菜单里填写的路径名一致
tags:
- java

这是我的第一个项目文章内容…
第三步：清理并重启
修改完后，必须重新生成一遍才能生效。

在终端执行：

Bash

hexo clean
hexo g
hexo s
现在，再次点击菜单里的 “project”，或者访问 http://localhost:4000/categories/project/，你应该能看到页面了，页面里会列出刚才那篇“我的项目演示”文章。

静态博客最常用且免费的部署方式是使用 GitHub Pages 或 Vercel/Netlify。

安装部署插件：

Bash

npm install hexo-deployer-git –save
配置部署信息： 修改主配置文件 _config.yml：

YAML

文件: blog/_config.yml

deploy:
type: git

替换为您的 GitHub Page 仓库地址

repo: <您的GitHub Pages仓库URL>
branch: main # 或 gh-pages
一键部署：

Bash

hexo deploy # 简称 hexo d

配置 Git 用户身份

您只需按照提示，在您的 Windows 命令行（PowerShell 或 Command Prompt）中运行两条命令，设置您的全局 Git 用户名和电子邮件地址即可。

设置用户电子邮件： 将下面的 “you@example.com“ 替换为您自己的邮箱地址。

Bash

git config –global user.email “your_email@example.com“
设置用户名称： 将 “Your Name” 替换为您的名字或您希望显示的用户名。

Bash

git config –global user.name “Your Name”
